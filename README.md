In this project, I approached software design by first carefully planning how to recreate real-world objects using basic 3D primitives. In my design document, I broke down the marker, bowl, push pin, and rubber duck into combinations of cylinders, cones, spheres, torus shapes, and boxes. This process strengthened my ability to decompose complex objects into manageable components and think spatially about scale, rotation, and positioning in a 3D coordinate system. I followed a structured process of planning, building, testing, and refining, which helped ensure that each object was proportionate and visually accurate. This design strategy of breaking problems into smaller parts and documenting ideas before implementation is something I can apply to future software projects to improve organization and efficiency. When developing the program, I worked incrementally, building and positioning one object at a time while continuously adjusting transformations, textures, and material properties. Iteration played a major role, as many elements required multiple refinements to achieve proper alignment and realistic lighting. Throughout the milestones, my coding approach evolved from simply rendering objects to writing cleaner, more organized code and managing materials and textures more effectively. This project strengthened my understanding of computational graphics concepts such as transformation matrices, lighting models, and texture mapping. These skills enhance my educational foundation in computer science and will support my professional goals by preparing me to develop an interactive game application.
